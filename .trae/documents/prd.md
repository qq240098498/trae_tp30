## 1. 产品概述

家庭应急管理工具是一款面向家庭用户的本地化 Web 应用，帮助用户集中管理紧急联系人、家庭住址、个人健康信息（血型、过敏史、慢性病史、常用药物），并支持一键生成"紧急信息卡片"（文本格式），方便截图保存至手机锁屏或打印随身携带。

- 解决紧急情况下信息分散、难以及时获取的痛点
- 目标用户：所有家庭成员，尤其适合有老人、儿童或慢性病患者的家庭

## 2. 核心功能

### 2.1 用户角色

本应用为纯本地单用户工具，无需注册登录，数据存储在浏览器本地。

### 2.2 功能模块

1. **紧急联系人管理**：录入、编辑、删除联系人，按优先级排序
2. **家庭信息管理**：录入家庭住址
3. **健康信息管理**：录入血型、过敏史、慢性病史、常用药物
4. **紧急信息卡片生成**：一键汇总所有信息，生成可截图/打印的文本格式卡片

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|---------|---------|---------|
| 主页面 | 紧急联系人区域 | 展示联系人列表，支持新增/编辑/删除，按优先级（第一联系人、第二联系人等）排序显示 |
| 主页面 | 家庭信息区域 | 展示家庭住址，支持编辑 |
| 主页面 | 健康信息区域 | 展示血型、过敏史、慢性病史、常用药物，支持编辑 |
| 主页面 | 紧急信息卡片生成 | 一键生成文本格式紧急卡片，支持复制到剪贴板，卡片设计适合截图保存或打印 |
| 主页面 | 数据持久化 | 所有数据自动保存到 localStorage，刷新不丢失 |

## 3. 核心流程

1. 用户首次进入应用，看到空白状态引导
2. 用户依次录入紧急联系人（姓名、关系、手机号、备用电话、优先级）
3. 用户录入家庭住址
4. 用户录入健康信息（血型、过敏史、慢性病史、常用药物）
5. 用户点击"生成紧急信息卡片"，系统自动汇总所有信息生成格式化文本卡片
6. 用户可截图保存或复制文本

## 4. 用户界面设计

### 4.1 设计风格

- **主题色**：深红色（#C62828）作为主色，象征紧急与警觉；暖灰色（#F5F0EB）作为背景色，营造安心感
- **辅助色**：琥珀色（#FF8F00）用于优先级标记，深绿色（#2E7D32）用于健康信息
- **按钮风格**：圆角（8px），主按钮使用深红填充，次按钮使用描边风格
- **字体**：标题使用 Noto Serif SC（衬线体，传达正式与庄重），正文使用 Noto Sans SC（无衬线，清晰易读）
- **布局风格**：单页卡片式布局，分区清晰，信息层次分明
- **图标风格**：使用线条风格的 FontAwesome 或 Material Icons 图标
- **整体氛围**：专业、清晰、紧急时刻一目了然，但平时使用时不会令人焦虑

### 4.2 页面设计概述

| 页面名称 | 模块名称 | UI 元素 |
|---------|---------|---------|
| 主页面 | 紧急联系人区域 | 卡片式列表，每张联系人卡片显示优先级标签（红色/琥珀色）、姓名、关系、手机号图标、备用电话；底部"添加联系人"按钮 |
| 主页面 | 家庭信息区域 | 地址输入卡片，带地图图标装饰 |
| 主页面 | 健康信息区域 | 网格布局四个信息项（血型/过敏史/慢性病/药物），每项带图标和标签 |
| 主页面 | 紧急信息卡片 | 模态弹窗或独立区域，红白配色紧急卡片样式，顶部红色横条带"紧急信息"标题，内容分区块排版，底部提示"截图保存至锁屏" |

### 4.3 响应式设计

- 桌面端优先，最大宽度 800px 居中布局
- 移动端自适应：联系人卡片单列堆叠，健康信息网格改为单列
- 触控优化：按钮最小点击区域 44px，联系人卡片支持滑动操作

### 4.4 动效

- 页面加载时各区域依次淡入（stagger animation）
- 联系人添加/删除时使用过渡动画
- 紧急卡片生成时使用缩放+淡入动画，强调视觉冲击力
- 按钮 hover 状态使用微妙的上浮效果
